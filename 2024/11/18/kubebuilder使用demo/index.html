Kustomize 的作用、背景与原因

Kustomize 是什么？

Kustomize 是一个 声明式 Kubernetes 对象定制工具，它允许你在不修改原始 YAML 的情况下，通过 kustomization.yaml 文件来管理和组织 Kubernetes 资源。

核心概念

概念 说明
Base 基础配置集合（如 crd, rbac, manager）
Overlay 环境特定的修改集合（如 default 组合所有 base）
Kustomization kustomization.yaml 文件，定义如何组织和修改资源

项目中的 Kustomize 结构解析

Kustomize 的关键功能

1. 资源组织 (resources:)

2. 参数修改 (namePrefix:, namespace:)

3. 补丁应用 (patches:)

补丁示例（manager_webhook_patch.yaml）：

4. 标签和注解管理 (labels:)

自动为所有资源添加标签，便于选择器使用

为什么 Kubebuilder 使用 Kustomize？

原因 说明
环境管理 轻松支持 dev/staging/prod 不同配置
代码复用 Base 可被多个 Overlay 重用
无脚本 纯声明式，不需要模板语言
模块化 清晰的关注点分离
Kubectl 原生支持 kubectl apply -k 直接支持
易于调试 kustomize build 可查看最终生成的 YAML
