OpenSSL 支持的密钥类型和格式

OpenSSL 支持的密钥类型及格式对比表

下面详细列出了 OpenSSL 支持的 密钥类型 及 格式，并结合典型应用场景帮助你选择合适的组合。

  1. 密钥类型对比

密钥类型 算法描述 典型应用场景 命令示例
RSA 基于大整数分解，经典公钥加密算法 TLS/SSL 证书、HTTPS、SSH openssl genrsa -out rsa_key.pem 2048
ECC 椭圆曲线密码学（高效、短密钥） IoT、移动设备、TLS、区块链 openssl ecparam -genkey -name prime256v1 -out ec_key.pem
DSA 数字签名算法（基于离散对数，较老） 数字签名（如 DNSSEC） openssl dsaparam -genkey 2048 -out dsa_key.pem
DH Diffie-Hellman 算法，用于密钥交换 VPN、IPsec 加密通道 openssl dhparam -out dh_key.pem 2048
ED25519 高性能椭圆曲线签名算法（无需参数生成） SSH、TLS 1.3、敏捷签名应用 openssl genpkey -algorithm ED25519 -out ed25519_key.pem
X25519 专用于密钥交换的椭圆曲线算法（现代加密协议） ECDH 密钥交换、TLS 1.3 openssl genpkey -algorithm X25519 -out x25519_key.pem
GOST 俄罗斯国家标准算法（对称/非对称加密） 使用 GOST 的应用、数字签名 需要 OpenSSL GOST 扩展
SM2 中国国家标准（基于 ECC） 国密应用（如商用加密与签名） openssl genpkey -algorithm SM2 -out sm2_key.pem

  1. 密钥格式对比

格式 描述 适用场景 加密支持 命令示例
PEM 文本格式（Base64 编码），以 —–BEGIN 开头 TLS/SSL 证书、私钥存储 支持加密 openssl rsa -in rsa_key.pem -outform PEM -out private_key.pem
DER 二进制格式，适合嵌入式系统和低级传输 嵌入式系统、低级传输 不支持加密 openssl rsa -in rsa_key.pem -outform DER -out private_key.der
PKCS#8 标准化私钥格式，支持多种算法（如 RSA、ECC） 服务器端密钥存储 支持加密 openssl pkcs8 -topk8 -in ec_key.pem -out pkcs8_key.pem
PKCS#12 二进制打包格式，包含私钥和证书链（.p12/.pfx） 客户端证书、VPN、S/MIME 全文件加密 openssl pkcs12 -export -out cert.p12 -inkey ec_key.pem -in cert.pem
OpenSSH 专用于 SSH 的密钥格式 SSH 登录和密钥交换 支持加密 ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519
JWK JSON Web Key 格式，用于 Web 应用的密钥管理 OAuth2、JWT 加密 支持加密 使用专用工具或库生成

  1. 常见应用场景及推荐组合

场景 推荐密钥类型 推荐格式 命令示例
Web 服务器（Nginx/Apache） RSA 或 ECC PEM、PKCS#8 openssl pkcs8 -topk8 -in rsa_key.pem -out server_key.pem
IoT 设备 ECC（prime256v1） PKCS#8 openssl ecparam -genkey -name prime256v1 -out ec_key.pem
VPN 客户端证书 RSA 或 ECC PKCS#12 openssl pkcs12 -export -out vpn_cert.p12 -inkey ec_key.pem -in cert.pem
SSH 登录 ED25519 或 RSA OpenSSH ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519
TLS 1.3 密钥交换 X25519 PKCS#8 openssl genpkey -algorithm X25519 -out x25519_key.pem
国密应用（SM2 签名） SM2 PKCS#8 openssl genpkey -algorithm SM2 -out sm2_key.pem

  1. 实现案例

  2. 生成 RSA 密钥并转换为 PKCS#8

openssl genrsa -out rsa_key.pem 2048
openssl pkcs8 -topk8 -in rsa_key.pem -out pkcs8_key.pem -nocrypt

  1. 生成 ECC 私钥并打包为 PKCS#12

openssl ecparam -name prime256v1 -genkey -out ec_key.pem
openssl req -new -x509 -key ec_key.pem -out cert.pem -days 365
openssl pkcs12 -export -out cert.p12 -inkey ec_key.pem -in cert.pem

  1. 生成 ED25519 密钥用于 SSH 登录

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519

  1. 总结与建议

    • RSA：适合兼容性要求高的应用（如 Web 服务器）。
    • ECC：更高效，适合资源受限设备和现代应用。
    • ED25519：轻量级且高性能，适合 SSH 和敏捷加密协议。
    • SM2：适用于国密标准要求的应用。

根据应用需求选择适当的密钥类型和格式，既能保证系统安全，又能提高效率与兼容性。
