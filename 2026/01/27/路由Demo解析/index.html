路由Demo解析

发表于 2026-01-27 阅读次数：
本文字数： 阅读时长 ≈ NaN:aN

时序流程图（Mermaid）

一、“总纲式结论”

跨三层通信的本质是：

  • IP 地址：端到端不变（逻辑寻址）
  • MAC 地址：逐跳改变（物理转发）
  • ARP：只解决“下一跳是谁”，而不是“最终是谁”

二、拓扑与角色明确

拓扑如下（你给的）：

  • Host1 的默认网关：Router A
  • Host2 直连在 Router C 的 LAN 上
  • 路由器之间都是三层转发
  • 终端和路由器之间是二层通信

三、完整路由过程（按步骤拆解）

阶段 1：Host1 决定“我该把包交给谁”

Step 1：Host1 判断目的 IP 是否本地

  • Host1 拿到 Host2 的 IP

  • 用自己的：

    • IP
    • 子网掩码

  • 计算后发现：
    Host2 不在本地子网

👉 结论：必须交给默认网关（Router A）

Step 2：Host1 通过 ARP 找默认网关的 MAC

  • 查看 ARP 表：

    • 有 Router A MAC → 直接用
    • 没有 → ARP Broadcast

  • 得到：

    • Router A（LAN 口）的 MAC

Step 3：Host1 封装第一个以太网帧

此时数据包状态是：

字段 值
Source IP Host1 IP
Dest IP Host2 IP
Source MAC Host1 MAC
Dest MAC Router A（LAN 口）MAC

👉 关键点：
Host1 不知道 Router B / C / Host2 的 MAC，也不需要知道。

阶段 2：Router A → Router B

Step 4：Router A 收到帧（L2 → L3）

Router A 做三件事：

  1. 看 目的 MAC → 是自己 → 收包

  2. 剥掉二层头，检查 目的 IP

  3. 查路由表：

    • Host2 不直连
    • 找到下一跳：Router B

Step 5：Router A 通过 ARP 找 Router B 的 MAC

  • ARP 查询：

    • IP：Router B（A-B 链路上的 IP）
    • 得到 Router B 接口 MAC

Step 6：Router A 重新封装帧并转发

新的帧：

字段 值
Source IP Host1 IP
Dest IP Host2 IP
Source MAC Router A（发往 B 的接口 MAC）
Dest MAC Router B（面向 A 的接口 MAC）

👉 IP 没变，MAC 换了

阶段 3：Router B → Router C（同理）

Router B 做的事情和 Router A 完全一样：

  1. 收到帧（MAC 是自己）
  2. 解封装
  3. 查路由表
  4. 下一跳是 Router C
  5. ARP 找 Router C MAC
  6. 重新封装并转发

新的帧：

字段 值
Source IP Host1 IP
Dest IP Host2 IP
Source MAC Router B（面向 C 的接口 MAC）
Dest MAC Router C（面向 B 的接口 MAC）

阶段 4：Router C → Host2（最后一跳）

Step 7：Router C 发现目的 IP 是本地直连

  • 查路由表：

    • Host2 IP 在本地 LAN

  • 不再需要“下一跳路由器”

Step 8：Router C ARP 查询 Host2 的 MAC

  • ARP：

    • IP：Host2
    • 得到 Host2 MAC

Step 9：Router C 发送最终帧

最终帧：

字段 值
Source IP Host1 IP
Dest IP Host2 IP
Source MAC Router C（LAN 口 MAC）
Dest MAC Host2 MAC

Step 10：交换机（二层）转发

  • 交换机只看 Destination MAC
  • 查 MAC 表
  • 发给 Host2 所在端口

👉 Host2 收到数据，去程完成

四、一路上哪些东西「一直不变」，哪些「一直在变」

✅ 一直不变的

  • Source IP：Host1
  • Destination IP：Host2

🔄 每一跳都会变的

  • Source MAC
  • Destination MAC

🧠 决策逻辑

  • IP 决定“去哪里”
  • MAC 决定“下一跳交给谁”

五、为什么不能“全网 ARP”（你原文最后的点睛）

如果没有路由器：

  • 所有设备在一个广播域
  • 每一次通信都 ARP 全网
  • 广播风暴
  • 网络直接瘫痪

👉 路由器的核心价值：

  • 切割广播域
  • 限制 ARP 的作用范围
  • 让 Internet 成为可能

六、一句“工程师级总结”

三层决定方向，二层负责搬运。
路由器每一跳只关心：
“下一步我该把包交给谁？”
