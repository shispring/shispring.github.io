pprof常用操作

pprof 交互模式最佳实践（命令速查表）

基础命令

命令 说明 使用场景
top 20 按 flat 值展示前 20 个函数 快速定位占内存/CPU 最多的函数
top 20 -cum 按 cum（累积）值排序 定位调用链上层的”源头”函数
list <函数名> 展示函数逐行的分配/CPU 占用 精确到哪一行代码造成的分配（**需要编译时保留符号，不能用 -s -w**）
web 生成调用图并打开浏览器 需要 Graphviz + 浏览器，服务器上一般不可用
svg 生成 SVG 输出到 stdout go tool pprof -svg file.pb.gz > out.svg
png 生成 PNG 图片 同上，输出重定向到 .png
pdf 生成 PDF 文档 适合存档分析报告

过滤与聚焦

命令 说明 使用场景
focus=<正则> 只展示匹配的函数 例如 top 20 -focus=informer
ignore=<正则> 排除匹配的函数 例如 top 20 -ignore=runtime
tags 展示 profile 中的标签信息 查看 label 维度
tree 以树形展示调用关系 理解调用链层级
traces 展示所有采样的完整栈 深入分析某类分配的完整调用路径

切换分析类型（核心技巧）

命令 说明 使用场景
sample_index=inuse_space 当前堆上驻留的内存量 默认值，排查”现在哪些对象在占内存”
sample_index=inuse_objects 当前堆上驻留的对象数量 排查大量小对象（如 string、slice header）
sample_index=alloc_space 从启动到现在累计分配的内存量 排查高频分配热点（即使对象已释放也能看到）
sample_index=alloc_objects 从启动到现在累计分配的对象数量 排查 GC 压力来源

在交互模式中输入 o 可查看当前选项，sample_index=alloc_space 然后 top 20 即可切换。

对比分析（检测泄漏最有效）

步骤 命令 说明
1. 抓第一次快照 curl -o heap1.pb.gz http://IP:6060/debug/pprof/heap 基线
2. 等 10-30 分钟 — 等待内存增长
3. 抓第二次快照 curl -o heap2.pb.gz http://IP:6060/debug/pprof/heap 对比点
4. 差异分析 go tool pprof -base heap1.pb.gz heap2.pb.gz 只展示增量，找泄漏点
5. 查看增量 top top 20 增量最大的函数就是嫌疑对象

其他 profile 类型

Profile URL 用途
heap /debug/pprof/heap 内存分配（最常用）
goroutine /debug/pprof/goroutine goroutine 泄漏检查
profile /debug/pprof/profile?seconds=30 CPU 热点（采 30 秒）
allocs /debug/pprof/allocs 累积分配（类似 heap 的 alloc_space）
block /debug/pprof/block 阻塞分析
mutex /debug/pprof/mutex 锁竞争分析
trace /debug/pprof/trace?seconds=5 运行时 trace（用 go tool trace 打开）
