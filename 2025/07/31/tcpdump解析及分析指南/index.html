🔍 组合标志位含义

⚠️ RST 标志位深度解析

RST (Reset) 的含义：

  • 立即终止 TCP 连接
  • 释放连接占用的资源
  • 不经过正常的四次挥手过程

RST 产生的常见原因：

  1. 端口不可达

  2. 防火墙拒绝

  3. 应用程序异常关闭

  4. 超时重置

典型场景分析

场景1：正常 HTTP 连接建立

分析要点：

  • ✅ 三次握手完整 (SYN → SYN+ACK → ACK)
  • ✅ 序列号正确递增 (1000 → 1001, 2000 → 2001)
  • ✅ 响应时间正常 (0.5ms)

场景2：连接被拒绝

分析要点：

  • ❌ 收到 RST 标志，连接被拒绝
  • 🔍 可能原因：端口未监听、防火墙阻止、服务未启动

场景3：连接超时重传

分析要点：

  • ⚠️ 重复的 SYN 包表示重传
  • 🔍 可能原因：网络延迟、服务器负载高、丢包

场景4：数据传输中的重传

分析要点：

  • ⚠️ 相同序列号的重复包表示重传
  • 🔍 可能原因：ACK 丢失、网络拥塞、接收方处理慢

场景5：异常连接终止

分析要点：

  • 💥 服务器主动 RST，应用层问题
  • 🔍 可能原因：程序崩溃、资源耗尽、业务逻辑错误

📚 知识点总结

TCP 状态机理解

  • LISTEN: 服务端等待连接
  • SYN_SENT: 客户端发起连接等待响应
  • SYN_RECEIVED: 服务端收到SYN等待ACK
  • ESTABLISHED: 连接建立，数据传输状态
  • FIN_WAIT_1/2: 主动关闭方的等待状态
  • CLOSE_WAIT: 被动关闭方等待应用程序关闭
  • TIME_WAIT: 主动关闭方的最终等待状态

负载均衡器行为模式

  • 4层LB: 基于IP+端口转发，维持连接状态
  • 7层LB: 解析应用协议，可以路由和修改内容
  • 健康检查: 定期探测后端服务可用性
  • 会话保持: 确保同一用户的请求到达同一后端

性能优化要点

  • 连接复用: 减少TCP握手开销
  • Keep-Alive: HTTP连接保持
  • 压缩传输: 减少网络传输量
  • 缓存策略: 减少不必要的网络请求
