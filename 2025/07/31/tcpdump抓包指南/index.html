抓包前的准备工作

1. 问题定义阶段

在开始抓包前，必须明确以下问题：

  • 🔍 具体现象：连接超时？响应慢？数据包丢失？
  • 📍 问题范围：客户端？服务端？中间网络设备？
  • ⏰ 发生时间：持续发生？间歇性？特定时间段？
  • 🌐 网络拓扑：了解数据包的完整传输路径
  • 📊 基线数据：正常情况下的网络表现如何？

常见网络问题场景

🚨 场景分类表

场景类型 典型问题 关注重点 预期结果
连接建立问题 连接超时、拒绝连接 TCP 三次握手 SYN, SYN-ACK, ACK
性能问题 响应慢、吞吐量低 延迟、重传、窗口大小 RTT, 重传率
DNS 解析问题 域名无法解析 DNS 查询和响应 Query/Response 匹配
HTTP/HTTPS 问题 4xx/5xx 错误、SSL 握手失败 应用层协议 HTTP 状态码、TLS 握手
防火墙/安全问题 包被丢弃、端口不通 ICMP 消息、RST 包 拒绝原因分析
负载均衡问题 请求分布不均、健康检查失败 多目标流量分析 负载分布模式

tcpdump 命令参数详解

基础语法

核心参数

参数 作用 示例
-i 指定网络接口 -i eth0, -i any
-w 写入文件 -w capture.pcap
-r 读取文件 -r capture.pcap
-c 捕获包数量 -c 1000
-n 不解析主机名 -n
-nn 不解析主机名和端口名 -nn
-v/-vv/-vvv 详细程度 -vv
-s 每个包的捕获长度 -s 65535 (完整包)
-X 十六进制和ASCII显示 -X
-A ASCII 显示 -A
-t 不显示时间戳 -t
-tt 显示精确时间戳 -tt

TCP 连接分析

  • SYN, SYN-ACK, ACK 三次握手完整
  • 序列号和确认号正确递增
  • 无异常 RST 包
  • 窗口大小合理
  • 无频繁重传

HTTP 分析

  • 请求方法和 URL 正确
  • HTTP 状态码符合预期
  • 响应时间在合理范围
  • Content-Length 与实际内容匹配
  • 无异常重定向

DNS 分析

  • 查询和响应配对正确
  • 响应时间 < 100ms
  • 无 NXDOMAIN 错误
  • DNS 服务器可达

⚠️ 注意事项

1. 性能影响

  • 大流量环境下抓包会消耗 CPU 和内存
  • 使用适当的过滤器减少数据量
  • 监控磁盘空间，避免写满系统盘

2. 安全考虑

  • 抓包文件可能包含敏感信息
  • 及时删除不需要的抓包文件
  • 在生产环境抓包需要额外审批

3. 法律合规

  • 确保有权限抓取网络流量
  • 遵守公司网络监控政策
  • 不要抓取他人隐私数据
