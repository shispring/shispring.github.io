IRQBALANCE_BANNED_CPUS 掩码含义详解

基本概念

IRQBALANCE_BANNED_CPUS 是 IRQ Balance 服务的配置参数，用于指定禁止IRQ Balance 自动分配中断的 CPU 核心。

位值含义

关键理解：

  • 0 bit = 允许使用 - IRQ Balance 可以将中断分配到这个 CPU
  • 1 bit = 禁止使用 - IRQ Balance 不会将中断分配到这个 CPU

示例说明

假设一个 8 核系统 (CPU 0-7)：

转换为十六进制: 4c

这意味着：

  • CPU 0,1,4,5,7 - 允许 IRQ Balance 分配中断 (bit=0)
  • CPU 2,3,6 - 禁止 IRQ Balance 分配中断 (bit=1)

实际应用场景

1. 网卡中断优化

2. NUMA 优化

3. 应用程序 CPU 隔离

我们脚本中的逻辑

在优化后的 irqbalance.sh 脚本中：

脚本目标

  • 根据网卡所在的 NUMA 节点，选择一部分 CPU 核心专门处理网卡中断
  • 将其余 CPU 核心标记为”禁止”，让它们专注于应用程序处理

处理流程

  1. 获取允许的 CPU (ALLOWED_CPUS) - 网卡可以使用的 CPU
  2. 计算禁止的 CPU (BANNED_CPUS) - 其余所有 CPU
  3. 生成掩码 - 将 BANNED_CPUS 对应的位设置为 1

总结

  • IRQBALANCE_BANNED_CPUS 是一个禁用掩码
  • 每个 bit 对应一个 CPU：0=允许，1=禁止
  • 目的是控制 IRQ Balance 的中断分配策略
  • 合理配置可以优化 NUMA 亲和性和 CPU 隔离
