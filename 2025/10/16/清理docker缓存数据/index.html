清理docker缓存数据

发表于 2025-10-16 阅读次数：

现象：构建镜像设备无空间

🧭 一、总体分析

类型 占用 可回收 说明
Images 13.21 GB 10.8 GB (81%) 很多旧镜像未使用
Containers 24 MB 100% 可回收 停止容器未清理
Local Volumes 2.96 GB 99% 可回收 旧卷未使用
Build Cache 85.73 GB 85.73 GB (巨量) Docker build 中产生的大量中间层缓存

🧹 二、分级清理策略

✅ 建议分三步执行，从“安全”到“彻底”。

1️⃣ 安全清理（释放最多空间，低风险）

交互时输入 y 确认即可，通常能释放 80GB 以上空间。
不影响现有镜像或运行中的容器。

2️⃣ 深度清理（删除所有未使用的镜像、容器、卷）

含义：

  • -a：删除所有未被容器使用的镜像（包括悬空镜像 <none>）。
  • --volumes：删除未被挂载的卷。
  • system prune：同时清理 network / container / build cache。

💡 慎用
此命令会删掉所有旧镜像，下次构建会重新拉取（时间成本较高，但可彻底释放空间）。
