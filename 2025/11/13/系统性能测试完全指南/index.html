Linux系统性能测试完全指南 🚀

一、截图命令分析

📊 系统状态概览

解析：

  • 12:49:30 - 当前系统时间
  • up 7 days, 1:57 - 系统运行了7天1小时57分钟
  • 3 users - 3个用户登录
  • load average: 12.87, 10.83, 7.73 - 系统负载（1分钟、5分钟、15分钟平均值）
    • 💡 这个值很高！ 对于一个CPU核心，理想值应该 < 1.0
    • 如果你有20核CPU，12.87意味着60%的负载，尚可接受
    • 如果只有8核，说明系统严重过载（160%负载）

💾 内存使用情况

解析：

  • 物理内存： 192GB总量
    • 已用：124GB (65%)
    • 空闲：40GB (21%)
    • 缓存：27GB (14%)
    • 可用内存：50GB （包含可回收的缓存）
  • Swap交换分区： 4GB
    • 已用：203MB (5%) - 正常，说明内存压力不大

评估： ✅ 内存使用率65%，尚有余量，健康状态

🔥 进程分析

从截图中看到大量 stress-ng 和 dd 命令：

1. stress-ng 命令

含义：

  • --vm 3 - 启动3个虚拟内存压测进程
  • --vm-bytes 28.05G - 每个进程分配28GB内存
  • --vm-method all - 使用所有内存压测方法（读写、复制、排序等）
  • --vm-hang 0 - 不挂起进程
  • --timeout 0 - 无限期运行（直到手动停止）

目的： 对内存进行压力测试

2. stress-ng CPU 测试

含义：

  • --cpu 21 - 启动21个CPU压测进程
  • --cpu-method matrixprod - 使用矩阵乘法算法（高强度计算）
  • --timeout 0 - 持续运行

目的： 对CPU进行满负载压力测试

3. dd 磁盘测试

含义：

  • if=/dev/zero - 输入源：无限产生0字节的设备
  • of=/dev/shm/memhog_5 - 输出到共享内存（RAM磁盘）
  • bs=1G - 块大小1GB
  • count=5 - 写入5个块（总共5GB）
  • iflag=fullblock - 确保读取完整块

目的： 测试内存写入性能（这里测试的是RAM，不是磁盘）

4. ping 命令

含义：

  • -f - 洪水ping（尽可能快地发送数据包）
  • -s 1 - 数据包大小1字节
  • 127.0.0.1 - 本地回环接口

目的： 测试网络栈和本地网络性能

八、性能测试注意事项 ⚠️

🔴 关键警告

  1. 生产环境慎用

    • stress-ng 会造成真实负载，可能影响业务
    • 建议在非高峰期或测试环境执行
    • 先小范围测试，再逐步增加强度

  2. 磁盘测试风险

    • dd 和 fio 会写入大量数据
    • 确保有足够磁盘空间
    • 避免在系统盘根目录测试

  3. 内存测试风险

    • 分配过多内存可能触发OOM Killer
    • 建议不超过可用内存的80%
    • 注意监控swap使用率

  4. 监控数据保存

    • 所有测试应保存日志，便于后续分析
    • 建议使用 tee 命令同时输出和保存

九、性能基准值参考

CPU

  • 负载平均值： 应 < 核心数
  • %idle： 正常应 > 20%
  • %iowait： 应 < 10%（高则磁盘瓶颈）

内存

  • 可用内存： 应 > 总量的20%
  • Swap使用： 应 < 10%
  • si/so： 应接近0（频繁swap说明内存不足）

磁盘

  • %util： 应 < 80%
  • await： SSD < 10ms，HDD < 20ms
  • IOPS： SSD > 10000，HDD > 100

网络

  • 带宽利用率： 应 < 80%
  • 丢包率： 应 < 0.1%
  • 延迟： 局域网 < 1ms，公网 < 100ms
