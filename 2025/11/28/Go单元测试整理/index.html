🚀 常用命令速查表

场景 命令 说明
快速测试 go test ./... 运行所有包的测试
详细输出 go test -v ./... 显示每个测试函数名
特定测试 go test -run TestName ./... 只运行匹配的测试
特定包 go test -v ./pkg/util/color 测试指定包
设置超时 go test -timeout 30s ./... 30秒超时
竞态检测 go test -race ./... 检测数据竞争
代码覆盖率 go test -cover ./... 显示覆盖率百分比
覆盖率文件 go test -coverprofile=cov.out ./... 生成覆盖率文件
覆盖率HTML go tool cover -html=cov.out HTML格式报告
基准测试 go test -bench=. ./... 运行所有基准测试
基准+内存 go test -bench=. -benchmem ./... 基准+内存统计
并行运行 go test -parallel 4 ./... 4个并发运行
运行N次 go test -count 3 ./... 运行3次
跳过长测 go test -short ./... 跳过耗时测试
显示日志 go test -v -run Test ./... 显示t.Log输出

🐛 常见问题

Q: 测试文件在哪里？

A: 与源文件在同一目录，以 _test.go 结尾

Q: 测试函数为什么不运行？

A: 检查是否符合命名规范

Q: 如何跳过某个测试？

A: 使用 t.Skip() 或 t.SkipNow()

Q: 如何设置测试超时？

A: 使用 -timeout 标志

Q: 为什么测试输出看不到 t.Log？

A: t.Log() 只在失败或使用 -v 时输出

Q: 如何运行子测试？

A: 使用 t.Run()

Q: 基准测试输出的数字什么意思？
