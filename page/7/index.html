关于FQDN

发表于 2025-07-07
本文字数： 阅读时长 ≈ NaN:aN

这是个很有技术深度的问题。在 Kubernetes 的网络架构中，FQDN（Fully Qualified Domain Name，完全限定域名） 是集群内部和外部通信中非常关键的一环。你提到的 FQDN 和 RQDN（可能是 Relative Qualified Domain Name 的意思，见下文）概念，都可以围绕 DNS 来系统理解。

✅ 一、什么是 FQDN？

FQDN（Fully Qualified Domain Name） 是指一个完全的、绝对的 DNS 名字，它从主机名开始，一直到根域（.）结束。

🔹 举例：

名称 说明
my-service.default.svc.cluster.local. ✅ FQDN，在 K8s 中完整的 service 解析域名
my-service.default.svc.cluster.local ✅ 几乎等价（根 . 可省略）
my-service ❌ 不是 FQDN，属于相对域名（可解析依赖 search 域）

✅ 二、Kubernetes 中的 FQDN 使用场景

✅ 三、FQDN 与非 FQDN 的区别

类型 示例 是否全局唯一 是否依赖 search 域 是否推荐
FQDN nginx.prod.svc.cluster.local ✅ 是 ❌ 不依赖 ✅ 推荐
非 FQDN（短域名） nginx 或 nginx.prod ❌ 可能冲突 ✅ 依赖 ❌ 不推荐跨命名空间

✅ 四、使用 FQDN 的好处

  1. 跨命名空间通信稳定

    • 使用完整域名 svc.ns.svc.cluster.local 不依赖 DNS search path

  2. 更高的安全性和确定性

    • 避免访问到其他命名空间中重名的 service（比如 nginx）

  3. 跨集群 DNS 联通（例如 Service Mesh、多集群）

    • 多集群环境中可以借助 FQDN 显式指定服务源

  4. 自动证书绑定（如 Istio、cert-manager）

    • 证书签发常用 FQDN 作为 CommonName 或 SAN

HTTPS 工作流程详解

1. TCP 握手阶段

  • TCP SYN: 客户端向服务器发送同步请求，请求建立连接
  • TCP SYN + ACK: 服务器响应并确认连接请求
  • TCP ACK: 客户端确认，TCP 连接建立完成

这是标准的 TCP 三次握手，为后续的 TLS/SSL 通信建立可靠的传输层连接。

2. 证书验证阶段

  • Client Hello: 客户端发送支持的 TLS 版本、加密套件列表等信息
  • Server Hello: 服务器选择 TLS 版本和加密套件
  • Certificate: 服务器发送数字证书（包含公钥）
  • Server Hello Done: 服务器表示握手信息发送完毕

在这个阶段，客户端会验证服务器证书的有效性，包括：

  • 证书是否由可信 CA 签发
  • 证书是否在有效期内
  • 证书中的域名是否与访问的域名匹配

3. 密钥交换阶段

这是 HTTPS 安全性的核心部分：

非对称加密过程：

  • 客户端生成一个随机的 session key（会话密钥）
  • 使用服务器证书中的 公钥 对 session key 进行加密
  • 发送 Client Key Exchange 消息，包含加密后的 session key
  • 服务器使用自己的 私钥 解密获得 session key

握手完成：

  • Change Cipher Spec: 双方通知对方后续通信将使用协商好的加密参数
  • Finished: 使用新的加密参数发送握手完成消息

4. 数据传输阶段

对称加密通信：

  • 双方都拥有相同的 session key
  • 使用对称加密算法（如 AES）对数据进行加密/解密
  • 所有 HTTP 数据都通过这个加密通道传输

加密机制说明

为什么使用混合加密？

  1. 非对称加密（RSA/ECDH）：

    • 优点：密钥分发安全，无需预共享密钥
    • 缺点：计算开销大，速度慢

  2. 对称加密（AES）：

    • 优点：加密速度快，效率高
    • 缺点：密钥分发困难

HTTPS 的巧妙之处：

  • 用非对称加密安全地交换对称密钥
  • 用对称加密进行实际的数据传输
  • 既保证了安全性，又保证了效率

1. 证书管理

2. TLS 版本配置

3. 安全头配置

5. 开发环境配置

7. 安全检查清单

  • ✅ 使用 TLS 1.2 或更高版本
  • ✅ 禁用不安全的加密套件
  • ✅ 启用 HSTS
  • ✅ 配置正确的证书链
  • ✅ 定期更新证书
  • ✅ 监控证书过期时间
  • ✅ 使用强加密算法（AES-256）
  • ✅ 启用 Perfect Forward Secrecy
